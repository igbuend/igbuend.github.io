Integrate git with the Hugo build process

Keep track of your last modification date with Git

HOWTO

Integrate git with the Hugo build process

Working Hard is not Smart

I believe most Hugo authors use Git as source control. This is a smart move and makes recovering from mistakes a breeze. in the case of a remote repository Git also provides a rudimentary back-up solution.

Why not take this a bit further and automate some of the configurable parameters of Hugo, relieving you from the task of updating the same item (the last modification date of a post) over and over?

If you modify a post, but forget to manually alter the lastmod setting in the Hugo frontmatter, you risk that search engines might not pick up your change. Since that happens a lot to me, the only good solution is to really forget about it by automating it.

As usual with Hugo you are not alone. Bigger minds have come up with a solution, as long as you use Git as a repository for your code.

If you are not familiar with Git, maybe read about using Bitbucket as a free but private Git repository first, and then come back to this howto.

A bit of preparation

First, include the following section in your config.toml:

[frontmatter]
    date = ["date", "publishDate", "lastmod"]
    lastmod = [":git", "lastmod", "date", "publishDate"]
    publishDate = ["publishDate", "date"]
    expiryDate = ["expiryDate"]

This is the default Hugo configuration,but I prefer it when it is made explicit in the configuration.

Look at the definition of lastmod, this is based on the value in the Git repository, and then will go to the definitions in the frontmatter of individual pages.

We prefer using Git, since it has the correct last modification date. Alternatively, authors or editors might forget to update the last modification date.

One more modification to config.toml is needed. At the start add the following line:

enableGitInfo = true

That’s it. Easy, isn’t it?

A bit of cleanup

Although the above will do the trick (lastmod defined by Git), authors might get confused if lastmod is still part of the Hugo frontmatter.

To avoid this confusion delete all lastmod lines in your Hugo archetypes.
